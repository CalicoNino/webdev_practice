Part 1

  • Description: This section consists of the Online Bookmarking portion of the Website. The User can either Login or SignUp then the user can add, edit and delete valid urls. In addition, they can view the top liste bookmarks on all pages.
  • Analysis: I used a one to many relationship between the users and the bookmarks in order to map out the database where the username is the foreign key for the Bokkmark Entity. Used PHP for login and logi validity. Also, I used jquery to setup functions and read from HTML. The reason behind this and not using vanilla javascript is because I'm a native Javascript/ReactJS user so I wanted to test out jquery as it is heavily used around the world. Also I attempted to use PHP session to control active login by the user, however it kept giving me errors so I based my solution to matching cookies saved when user logs in and removed when the user logged out. This solution was implemented by W3Schools (which I referenced in the code). I used PHP to cmmunicate with the database and call in queries which resulted in being very easy and efficient.
  • Documentation: I added comments to the code where comments seemed to be needed. Most documentation is referencing to within the code itself.
  • User Guide: To begin, follow the following steps:
    • Click on Part 1 on the global navbar.
    • You can always view the Top Ten Bookmarked Pages.
    • SignUp by clicking sign up and inputing a username and password.
    • You will be automaitcally logged in. However, you can alternatively login if a user account already created.
    • User can then add a bookmark and if an error occurs, the user will be displayed a message.
    • After a bookmarkis added, the user can edit or delete that bookmark.
    • When done, the user can log out or simply exit the screen. The user will be automaically logged in at their next visit since they haven't logged out.
